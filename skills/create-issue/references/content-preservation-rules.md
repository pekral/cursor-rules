# Content Preservation Rules

## Always

- Preserve the **original task text exactly** as provided
- Never rewrite, summarize, or modify the content
- Only improve **formatting and structure** for readability
- Automatically **assign the issue to the current user**
- Create the issue **using installed CLI tools available in the environment**
- Generate the **issue title from the first line of the task description**

## Never

- Change wording of the original task
- Add new requirements that were not in the original
- Remove information from the original task
- Remove any tasks in the issue tracker
- Merge, split, or reorder the original content in a way that alters meaning
