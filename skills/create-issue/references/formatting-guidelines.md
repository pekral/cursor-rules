# Formatting Guidelines

## Allowed Formatting Changes

Improve readability using markdown formatting. The following changes are permitted:

- **Headings** — add section headers to organize content
- **Bullet lists** — convert inline lists to bullet points
- **Numbered lists** — convert sequential steps to numbered lists
- **Code blocks** — wrap code snippets, file paths, or commands in code blocks
- **Spacing** — add blank lines between sections for readability

## Constraint

Do not modify the meaning of the text. Formatting is cosmetic only.

## Issue Structure

Format the issue body like this:

```markdown
# Task

<original task text, with formatting improvements>

---

# Notes

This issue was automatically formatted for readability.
Original task content was preserved exactly.
```

## Section Breakdown

| Section | Content |
|---|---|
| **Task** | The original task description with improved formatting |
| **Notes** | A short disclaimer that the issue was auto-formatted |
