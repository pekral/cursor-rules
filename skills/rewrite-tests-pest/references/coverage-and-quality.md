# Coverage and Quality Requirements

## Coverage

- Tests must achieve **100% coverage** of the code under test
- After writing tests, verify coverage and add missing cases if needed

## Quality Checks

- After creating or modifying tests, check that they are not flaky
- Verify that tests are functional and follow the project's test-writing guidelines
- Ensure all tests pass before considering the rewrite complete

## Post-Completion

- If the changes can be tested according to `@skills/test-like-human/SKILL.md`, run those tests as well
