# Output Quality Checklist

## The final proposal must satisfy all of the following criteria

- [ ] **Concrete** — describes a specific change, not a vague direction
- [ ] **Measurable** — includes at least one way to verify success (metric, test, observable behavior)
- [ ] **Actionable** — provides a minimal implementation plan that can be started immediately
- [ ] **Singular** — contains exactly one proposal, not multiple alternatives
- [ ] **Risk-aware** — lists key risks with mitigations
- [ ] **Scoped** — targets the smallest safe iteration, not a multi-sprint epic
- [ ] **Rollback-ready** — includes rollout/rollback notes

## Common mistakes to avoid

- Proposing an entire platform rewrite instead of an incremental improvement
- Listing benefits without acknowledging risks
- Providing a plan so vague it cannot be estimated or started
- Combining multiple independent ideas into one proposal
- Ignoring existing patterns and conventions in the codebase
