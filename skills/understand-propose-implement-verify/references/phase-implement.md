# Phase: Implement

## Goal

Execute the proposed solution end-to-end, keeping changes focused and aligned with project conventions.

## Required activities

- Implement exactly what was proposed — no scope creep
- Follow existing project conventions for code style, naming, structure
- Add or update tests for all changed behavior
- Invoke relevant project skills where identified in the Propose phase
- Keep commits atomic and well-described

## Quality checklist

- [ ] Changes are focused on the proposed solution only
- [ ] No unrelated modifications were introduced
- [ ] Tests cover the new or changed behavior
- [ ] Code follows existing project conventions
- [ ] Existing skills were reused where applicable

## Rules

- Do NOT deviate from the proposed solution without re-entering the Propose phase
- Do NOT skip tests — every behavioral change needs verification
- If an unexpected issue arises during implementation, pause and reassess (go back to Understand if needed)
- Keep changes deterministic and reproducible
