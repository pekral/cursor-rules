# Phase: Understand

## Goal

Build a complete, verified picture of the problem before any solution work begins.

## Required activities

- Read the full assignment: description, comments, linked issues, attachments
- Download and inspect all referenced resources using CLI tools or MCP (never a web browser)
- Classify the request into one of: bug, feature, refactor, review, docs, infra
- Identify the affected scope: files, modules, services, APIs
- List all explicit and implicit requirements

## Output

A short task checklist containing:
- Problem statement (in your own words, not copy-pasted)
- Request classification
- Assumptions and constraints
- Open questions that need clarification before proceeding

## Rules

- Do NOT propose solutions during this phase
- Do NOT make changes to code or configuration
- If critical information is missing, ask for clarification before moving on
- Separate verified facts from assumptions explicitly
