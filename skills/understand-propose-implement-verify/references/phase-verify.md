# Phase: Verify

## Goal

Confirm that the implementation is correct, complete, and free of regressions.

## Required activities

- Run all relevant fixers, linters, and formatters for the changed scope
- Run the test suite (at minimum the tests covering changed code)
- Verify that all requirements from the Understand phase are satisfied
- Check for regressions in related functionality
- Produce a final status report

## Verification checklist

- [ ] All tests pass (new and existing)
- [ ] Linters and formatters report no issues
- [ ] Every requirement from the task checklist is addressed
- [ ] No regressions were introduced
- [ ] Output quality meets project standards

## Rules

- Do NOT skip verification — every implementation must be verified
- If verification reveals a problem, go back to the appropriate phase (Understand, Propose, or Implement)
- Report all findings honestly, including any remaining risks or caveats
- Verification is not complete until the final status report is produced
